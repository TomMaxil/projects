# Online Supermarket System (ASP.NET Core MVC)

## Project Overview
Online Supermarket System is a web-based e-commerce application developed using ASP.NET Core MVC. 
The system provides separate panels for Guest, Customer, and Admin users.

This project was developed as an academic + self-learning project to understand real-world e-commerce workflows.

## Technologies Used
- ASP.NET Core MVC
- C#
- Entity Framework Core (Code First)
- SQL Server
- HTML, CSS, JavaScript
- Bootstrap
- Custom Authentication & Authorization

## User Roles & Features

### Guest Panel
- View products with category-wise filtering
- Dynamic search and sorting
- Category cards with product listing
- Brand slider section
- Product detail view

### Customer Panel
- User registration & login
- Add products to cart
- Place orders (dummy credit card simulation)
- View order history and order status
- Submit product reviews

### Admin Panel
- Dashboard overview
- CRUD operations for:
  - Products
  - Categories
  - Brands
- Manage users
- Manage orders (Pending, Shipping, Completed)

## Key Concepts Implemented
- MVC Architecture
- Model Binding & Validation
- Custom Authentication & Authorization
- Code First Migrations
- Role-Based Access Control
- Dashboard Reporting

## Purpose of Project
This project helped me understand:
- Real e-commerce business logic
- Multi-role system design
- Database relationships
- Secure application development

## Future Improvements
- Online payment gateway integration
- API-based mobile support
- Performance optimization

# Inventory Management System (ASP.NET Core MVC + API)

## Project Overview
Inventory Management System is a real-world inspired application developed using ASP.NET Core MVC.
It focuses on managing products, stock levels, sales, and automated notifications.

## Technologies Used
- ASP.NET Core MVC
- ASP.NET Core Web API
- Entity Framework Core
- SQL Server
- SignalR
- SMTP Email (Gmail)
- Toastr Notifications
- xUnit & Moq (Unit Testing)

## Core Features
- Product & category management
- Stock and minimum stock tracking
- Sale management with stock deduction
- Dashboard with low-stock count
- Email notifications for low stock
- Real-time alerts using SignalR
- RESTful APIs for product management
- Swagger API documentation

## Advanced Concepts Implemented
- Service Layer Architecture
- Dependency Injection
- SignalR Real-Time Communication
- SMTP Email Integration
- REST APIs
- Unit Testing with Moq
- Clean Code Practices

## API Capabilities
- Create, update, delete products
- Fetch products for mobile or frontend apps
- Swagger UI for testing APIs

## Purpose of Project
This project was built to:
- Learn enterprise-level architecture
- Understand inventory business rules
- Practice APIs for mobile integration
- Implement real-time systems

## Future Enhancements
- JWT Authentication
- Mobile App integration
- Role-based API security
